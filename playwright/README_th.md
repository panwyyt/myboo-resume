# 🎭 Playwright: พลังรวมศูนย์ของโลกออโตเมชัน

สวัสดีครับ! 👋 ยินดีต้อนรับสู่โลกของ Playwright ที่ผมสร้างขึ้นครับ

โปรเจกต์นี้คือการโชว์ศักยภาพที่ผมนำเอา **Playwright** มาใช้จัดการทั้งการทดสอบหน้าเว็บ (Web UI) และการทดสอบระบบข้อมูลหลังบ้าน (API) ให้อยู่รวมกันในที่เดียวครับ มันคือ "สุดยอดเครื่องมือตัวเดียวที่ทำได้ทุกอย่าง" สำหรับ QA ยุคใหม่จริงๆ

---

## 🌟 ทำไมผมถึงเลือก Playwright? (เครื่องมือเดียวคือที่สุด)

ก่อนหน้านี้ผมใช้ **Selenium** สำหรับ UI และ **Postman** สำหรับ API ซึ่งมันยอดเยี่ยมทั้งคู่ครับ แต่พอได้ลองใช้ Playwright... ผมก็ไม่เปลี่ยนใจอีกเลย:
-   **ความเร็ว:** รันแบบขนานได้ในตัวเอง (Parallel Execution) ทำให้ประหยัดเวลาสุดๆ
-   **ความเสถียร:** มีระบบ "Auto-wait" ในตัว ช่วยลดปัญหาเทสต์พังเพราะดึงข้อมูลไม่ทัน
-   **ความสะดวก:** ไม่ต้องสลับแอปไปมา โค้ด UI และ API ของผมอาศัยร่วมกันได้อย่างสมบูรณ์แบบครับ

---

## 🏗️ เส้นทางการทำงาน: Playwright Workflow ของผม

ผมตั้งใจออกแบบพอร์ตนี้เพื่อให้เห็นความหลากหลายในการนำไปใช้งานจริงครับ

### Step 1: ความมหัศจรรย์ของ Codegen
การสร้างเทสต์ไม่จำเป็นต้องเริ่มจากศูนย์เสมอไปครับ! ผมใช้ **Playwright Codegen** เพื่อบันทึกการกระทำของผมบนหน้าเว็บไซต์ และให้มันเขียนโค้ดออกมาให้โดยอัตโนมัติ ช่วยให้ผมตั้งต้นสคริปต์ได้ไวขึ้นมหาศาลเลยครับ

![Playwright Codegen ในการทำงาน](using-codegen.gif)
*(ผมแค่คลิกๆ เล่นๆ ไป... แล้วปล่อยให้ Playwright พิมพ์โค้ดให้ผมแบบ Real-time)*

### Step 2: จัดหนัก UI ด้วย Pattern แบบ POM
ผมทำระบบออโตเมชันบนแบบฟอร์มลงทะเบียนที่ซับซ้อน ครอบคลุมทั้งปุ่มตัวเลือก, ตารางวันที่, การอัปโหลดไฟล์ และรายการเลือกต่อเนื่อง (Cascading dropdowns) และเพื่อให้โค้ดอ่านง่าย แก้ไขสะดวก ผมจึงใช้โครงสร้างแบบ **Page Object Model (POM)** ครับ

![โครงสร้างโค้ดแบบ POM ในไฟล์เทสต์](pom-code-structure.gif)
*(แยกที่อยู่พิกัดปุ่มและลอจิกออกจากกัน เพื่อความเสถียรและความสวยงามของโค้ดครับ)*

### Step 3: API Testing (จัดเต็มครบ CRUD)
จะขาดเรื่องข้อมูลไปไม่ได้! ผมใช้ฟีเจอร์ `request` ที่ติดมากับ Playwright ในการจัดการข้อมูลแบบครบวงจร (GET, POST, PUT, DELETE) โดยไม่ต้องพึ่งห้องสมุดโค้ดจากที่อื่นเลยแม้แต่บรรทัดเดียวครับ!

![หน้าตาโค้ดเทส API ใน VS Code](api-testing.png)
*(หน้าตาโค้ด API พร้อมคอมเมนต์ภาษาไทยอธิบายละเอียดยิบในทุกขั้นตอน)*

## 🛠️ ระบบที่ผมใช้ (Tech Stack)

-   **Stack:** Playwright v1.58.2 + TypeScript + Node.js
-   **Pattern:** Page Object Model (POM) + Data-Driven Testing

```bash
# วิธีรันเบื้องต้น
npm install
npx playwright install
npx playwright test
```
