# 🚀 Postman: ถอดรหัสซอฟต์แวร์จาก "ข้างในสู่ข้างนอก"

ยินดีต้อนรับเข้าสู่อีกด้านของพอร์ตโฟลิโอ API ของผมครับ! 🌐

ในฐานะ QA เป้าหมายสูงสุดของผมคือการทำให้ซอฟต์แวร์ทำงานได้เป๊ะทุกระเบียดนิ้ว แต่ผมเชื่อว่าโปรดักต์ที่แข็งแกร่งจริงๆ ต้องเริ่มจาก "รากฐาน" ที่มั่นคงครับ นั่นคือเหตุผลที่ผมให้ความสำคัญกับการเทสชั้นข้อมูลหลังบ้าน (API) อย่างมาก เพราะถ้า "ท่อส่งข้อมูล" ของเราทำงานได้ดี ปลอดภัย และรวดเร็ว... ปัญหาหรือบั๊กนับไม่ถ้วนก็จะไม่มีวันหลุดรอดไปถึงมือผู้ใช้งานหน้าจอครับ!

ในส่วนนี้ ผมใช้ **Postman** เพื่อโชว์ให้เห็นว่าผมสามารถคุยกับ API โดยตรง แกะตรรกะเบื้องหลัง และวางระบบเทสอัตโนมัติได้อย่างรัดกุม—ตั้งแต่อุดมคติยังไม่ได้กลายเป็นโค้ดหน้าบ้านเลยด้วยซ้ำ!

---

## 🙏 ขอบคุณขุมทรัพย์ข้อมูลจำลอง

ผมขอขอบคุณพื้นที่ Sandbox ดีๆ ที่ช่วยให้ผมได้ฝึกฝนฝีมือและนำมาโชว์ในพอร์ตนี้ครับ:
-   **JSONPlaceholder:** สนามเด็กเล่นสำหรับการฝึกจัดการข้อมูลพื้นฐาน (CRUD) จนคล่องมือ
-   **Restful-Booker:** ระบบจองโรงแรมที่ซับซ้อนขึ้น ซึ่งช่วยให้ผมได้ฝึกฝนการจัดการ "รหัส Token" และมิติด้านความปลอดภัยครับ

## 🏗️ เส้นทางการทำงาน: API Testing Workflow ของผม

ผมแบ่งการเทสออกเป็น 3 ขั้นตอนสำคัญ เพื่อให้มั่นใจว่าระบบจะ "เอาอยู่" ในทุกมิติครับ

### Step 1: พื้นฐานต้องแม่น (CRUD Operations)
API ที่ดีต้องเริ่มจากการจัดการข้อมูลเบื้องต้นให้เป๊ะ! ผมเขียนคำสั่งตรวจสอบ (Assertions) เพื่อเช็คให้มั่นใจว่าทุุกครั้งที่ยิงคำขอไป... เซิร์ฟเวอร์จะตอบกลับมาด้วย Status Code ที่ถูกต้องและรูปแบบข้อมูลที่เป๊ะตามสั่ง 100%

  <img src="boo-basic-api/runResult2.png" width="48%" alt="รายละเอียดความแม่นยำ" />
</p>
*(สอบผ่านฉลุย ครอบคลุมการจัดการข้อมูลสเต็ปพื้นฐานทั้งหมด)*

### Step 2: อัพเกรดเป็นระบบอัตโนมัติ (Smart Scripting)
การเทสมือมันช้าไปครับ! ผมจึงอัดฉีดความสามารถของ **JavaScript** เข้าไปใน Postman เพื่อให้มัน "คิดเอง" ได้:
-   **การปั้นข้อมูล (Dynamic Data):** เขียนสคริปต์ให้ระบบเสกชื่อหรือราคาแบบสุ่มขึ้นมาทดสอบเอง
-   **ผลลัพธ์แบบห่วงโซ่:** สั่งให้ระบบ "เจาะ Token" จากคำขอ Login แล้วส่งต่อไปปลดล็อกงานชิ้นถัดไปได้ทันที... ไม่ต้องมานั่งก๊อปวางให้เสียเวลาครับ!

![Automated Tests with Scripts](boo-advance-api/boo-advanced-Restful-Booker-runResult.png)
*(การสกัด Token และการสร้างข้อมูลจองโรงแรมเลียนแบบด้วยสคริปต์รันออโต้แบบไร้รอยต่อ)*

### Step 3: ทดสอบขีดจำกัด (Performance)
ระบบที่รันผ่านคนเดียวได้... อาจจะล้มตายตอนเจอคนพันคนเหยียบกันพุ่งเข้ามา! ผมจึงกระหน่ำยิงการทดสอบประสิทธิภาพเพื่อดูว่า API ของเราจะยังนิ่งอยู่ไหมยามเจอความกดดันมหาศาล เพื่อรับประกันว่ารากฐานโปรเจกต์จะมั่นคงแข็งแกร่งที่สุดครับ

## ⚡ โบนัสสกิล: ความคุ้นเคยกับ Swagger (OpenAPI)

ส่วนสำคัญของชีวิต QA สาย API ก็คือ **Swagger** ที่เปรียบเหมือน "คู่มือการใช้งาน" ครับ ทริคสุดเทพของผมคือการไปกดยิงคำขอใน Swagger UI แล้วก๊อปปี้ **cURL command** มาโยนใส่ Postman (Import > Raw text)... มันจะช่วยกางทุกอย่างให้ครบในพริบตาเดียว! ช่วยประหยัดเวลาพิมพ์มือ และพุ่งตัวไปเริ่มจับบั๊กได้ไวขึ้นหลายเท่าเลยครับ!
