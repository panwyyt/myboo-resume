# 📱 Mobile Automation: ย้ายฐานการทดสอบสู่หน้าจอมือถือ

ยินดีต้อนรับสู่ส่วนของ Mobile Automation ในพอร์ตโฟลิโอของผมครับ! 🚀

หลังจากที่ผมได้โชว์การเทสบนเว็บไซต์ที่ใช้ Selenium ไปแล้ว คราวนี้ผมขยับมาท้าทายตัวเองด้วยสิ่งที่มีความยากขึ้นไปอีกขั้น นั่นคือ **แอปพลิเคชันบนมือถือ** ไม่ว่าจะเป็นแอปจาก App Store หรือ Google Play ทั้งหมดนี้ต้องอาศัย "พลังพิเศษ" ที่ต่างไปจากเดิมครับ

---

## 🌟 จุดเริ่มต้น: จาก Web สู่ Mobile

ถ้าเราเปรียบเทียบ Robot Framework เป็น "สมอง" เจ้าน้อง **Appium** ก็คือ "คนขับรถ" ที่มีความเชี่ยวชาญบนเส้นทางของ iOS และ Android มากเป็นพิเศษนั่นเองครับ ข่าวดีก็คือ เพราะผมมีพื้นฐานที่แน่นหนาใน Robot Framework อยู่แล้ว ผมจึงสามารถนำตรรกะเดิมมาใช้ต่อยอดได้เลย เพียงแค่ต้องเรียนรู้วิธี "กำราบมอนสเตอร์" ฝั่งมือถือเพิ่มเท่านั้นเองครับ

---

## 🧩 ความท้าทาย: ทำไมแอปมือถือถึงเทสต์ยากกว่า?

การทำ Automation บนมือถือยากกว่าบนเว็บพอสมควรครับ นี่คือเหตุผลว่าทำไมมันถึงท้าทายกว่า:

1.  **โลกเสมือนจริง:** เราไม่สามารถแค่เปิดหน้าเบราว์เซอร์ได้ง่ายๆ ครับ ผมต้องรัน "มือถือเสมือน (Virtual Phone)" ขึ้นมาทั้งเครื่องบนคอมพิวเตอร์! นั่นคือการเซ็ตอัป **Android Studio** เพื่อสร้าง Emulator และใช้ **Xcode** เพื่อสร้าง iPhone จำลอง (Simulator) ขึ้นมาครับ
2.  **กล้อง X-Ray:** คุณไม่สามารถ "คลิกขวา > Inspect" บนมือถือได้เหมือนหน้าเว็บ ผมจึงต้องใช้เครื่องมือที่ชื่อว่า **Appium Inspector** ซึ่งทำหน้าที่เหมือนเครื่อง X-Ray ช่วยให้ผมมองเห็น "กระดูก" (Locators) ที่ถูกซ่อนไว้ของแอป เพื่อที่ผมจะสั่งให้บอตไปกดปุ่มได้อย่างแม่นยำครับ

## 🤖 ภารกิจที่ 1: การเทสบน Android (การเช็คหมายเลข IMEI)

ในมิชชันนี้ ผมสั่งให้ระบบเปิดแอปโทรศัพท์ขึ้นมาโดยอัตโนมัติ เพื่อทำการกดปุ่มตัวเลขบนหน้าจอและเช็คหมายเลข IMEI ประจำเครื่องครับ

### Step 1: การสแกนระบบอย่างละเอียด
ผมใช้ Appium Inspector "ส่อง" ผ่านหน้าจอ Android และหาพิกัดที่แม่นยำที่สุดสำหรับปุ่มตัวเลขบนคีย์บอร์ด
![Android Inspector](android/running_test.png)

### Step 2: ปฏิบัติการด้วยความเร็วแสง
มาลองดูภาพตอนที่สคริปต์รันงานแบบอัตโนมัติกันครับ มันจะขยับและกดปุ่มได้เหมือนกับมีคนจริงๆ มาเป็นคนเล่นเป๊ะๆ เลยครับ
![Android Test Execution](android/automation_evidence.gif)

### Step 3: การันตีความสำเร็จ
พอระบบรันเทสต์เสร็จ Robot Framework ก็จะสร้างรายงานที่พ่นผลออกมาเป็นสีเขียวทั้งหมด ยืนยันว่าการเช็คหมายเลข IMEI สำเร็จลุล่วงครับ
![Android Test Report](android/test-report.png)

*(เกร็ดความรู้สำหรับสายเทค: สามารถดูสคริปต์ฉบับเต็มได้ที่นี่ครับ: [`android/test_mobile.robot`](android/test_mobile.robot))*

## 🍏 ภารกิจที่ 2: การเทสบน iOS (แอป Reminders)

ฝั่ง iOS มีกฎระเบียบของระบบที่ต่างจาก Android โดยสิ้นเชิง ผมเลือกเทสกับแอป **การเตือนความจำ (Reminders)** ที่ติดมากับเครื่อง iPhone เพื่อโชว์ทักษะการทำงานข้ามแพลตฟอร์ม (Cross-platform) ครับ

### Step 1: เตรียมสภาพแวดล้อมจำลอง
ผมตั้งค่า **Xcode iOS Simulator** ขึ้นมาเพื่อรันหน้าต่างเป้าหมาย การหาพิกัดบน iOS ต้องอาศัยทักษะการคิดวิเคราะห์ที่ต่างจากฝั่ง Android พอสมควรเลยครับ
![iOS Inspector](ios/running_test.png)

### Step 2: ขับเคลื่อนอย่างลื่นไหล
มาดูจังหวะที่บอตเปิดแอป Reminders และเข้าไปทำภารกิจตามที่วางแผนไว้ได้อย่างเป็นธรรมชาติสุดๆ กันครับ
![iOS Test Execution](ios/automation_evidence.gif)

### Step 3: บทสรุปสุดท้าย
ปิดภารกิจด้วยรายงานผลที่มีความละเอียดสูง ยืนยันว่าระบบเทสบน iOS ทำงานได้อย่างไม่มีที่ติครับ
![iOS Test Report](ios/test-report.png)

*(เกร็ดความรู้สำหรับสายเทค: สามารถดูสคริปต์ฉบับเต็มได้ที่นี่ครับ: [`ios/ios_basic_test.robot`](ios/ios_basic_test.robot))*
