## 🧐 รู้จักคู่หู Robot Framework & Selenium Library ให้ลึกซึ้งยิ่งขึ้น

ลองหลับตาจินตนาการดูสิครับ ว่าถ้าเรามีผู้ช่วยดิจิทัลที่ไม่มีวันเหน็ดเหนื่อย หน้าที่เดียวของเขาคือ คอยเปิดหน้าต่างเว็บ กดปุ่มที่กำหนด และกรอกข้อความตามสั่งแบบเป๊ะๆ ซ้ำๆ แบบไม่บ่นซักคำ นั่นแหละครับคือสิ่งที่ระบบนี้กำลังทำให้เรา!

- **Robot Framework**: ผมขอยกให้หมอนี่เปรียบเสมือนเป็น "สมองสั่งการ" มันคือเฟรมเวิร์กสาย Open-source สุดเทพที่อ่านง่ายเกินคาด แทนที่เราจะมัวเผชิญกับโค้ดโครงสร้างที่ซับซ้อน มันกลับอนุญาตให้เราเขียนคำสั่งเป็น 'ประโยคภาษาอังกฤษแบบบ้านๆ' (เรียกกันว่า Keyword-driven) *ข้อดีสุดจึ้ง:* โค้ดอ่านง่าย คนนอกสายโปรแกรมเมอร์ก็เข้ามาช่วยอ่านได้สบายมาก! แถมมันยังมาพร้อมระบบกดปุ่มพ่นสรุป Report สวยๆ ด้วย
- **Selenium Library**: เจ้านี่เปรียบเหมือน "แขนและมือ" ของระบบ มันคืออุปกรณ์เทสต์เว็บเบราว์เซอร์ที่จับเชื่อมต่อเข้าคู่ไปกับเจ้าตัว Robot Framework เพื่อยึดการควบคุมตัวหน้าเว็บ *ข้อดีสุดจึ้ง:* มันสื่อสารและเล่นไล่เคาะหน้าปุ่ม ดรอปดาวน์ และกล่องค้นหาได้เนียนกริบเหมือนกับมีคนเล่นนั่นแหละครับ แต่มันทำรวดเร็วและแม่นยำดุจสายฟ้าแลบเท่านั้นเอง!

เมื่อรวมตัวกันปุ๊บ พวกเขาก็ลบงานน่าเบื่อๆ ให้กลายเป็นความราบรื่นสไตล์ออโตเมชันที่เชื่อถือได้แบบทันที!

## 📂 ตัวโปรเจกต์: ภารกิจพิชิตหน้าฟอร์มที่ซับซ้อนสุดๆ

ในโฟลเดอร์ Repository ชิ้นนี้ คุณจะเจอกับตัวโปรเจกต์ตัวหนึ่งที่ถูกขนานนามให้ว่า `form-filling-page` 

เป้าหมายตรงนี้ดูดาดๆ แอบแฝงด้วยความท้าทายล้วนๆ ครับ: นั่นก็คือการปั้นสคริปต์เพื่อให้ระบบเข้าไปกรอกแบบฟอร์มยากๆ ที่เต็มไปด้วย กล่องข้อความ เมนู Dropdowns สารพัดตัวติ๊กเช็คบ็อกซ์ การบังคับใส่แนบไฟล์ จนยันกล่องเลือกวันเวลา (Date Picker) ระดับไดนามิค

### 1. สนามทดลองจริง

หากมองในสายตาคนทั่วไป มันก็จะดูเหมือนหน้าเว็บลงทะเบียนสมาชิกแบบงั้นๆ แต่ลองมาสวมแว่นสายนักเขียนสคริปต์ออโต้สิครับ นี่มันคือด่านสิ่งกีดขวางสารพัดรูปทรงอุปสรรค! ที่ต้องเจอกับรูปแบบโครงสร้าง HTML มั่วตั้วหลากหลายชนิด ที่บังคับให้จัดการไม่เหมือนกันเลย 

![The Testing Form UI](form-filling-page/other-data/testing-form-ui.png)

### 2. เขียนโค้ดควบคุมกันหน่อย

ก่อนที่ปาฏิหาริย์จะก่อเกิดขึ้น จะต้องมีการเขียนสคริปต์คำสั่งเอาไว้ก่อน การหยิบเอาภาษาตระกูล **Robot Framework** ยิ่งทำให้หน้าโค้ดโล่งสะอาด ดูเข้าใจง่ายคล้ายสมุดจด Check-list คำสั่งภาษาอังกฤษเลยครับ และดานล่างนี้คือหน้าตาเล็กๆ น้อยๆ ของโค้ดเบื้องหลังครับ:

![Test Script Code](form-filling-page/other-data/test-writing-code.png)

### 3. ชมฉากแอคชั่นกันได้!

ในยามที่สคริปต์พร้อมทำงาน มันก็จะพุ่งเข้ายึดครองหน้าจอ ขยับนู่น เติมนี่ พิมพ์ข้อความได้อย่างไร้จุดด่างพร้อย ขอบอกเลยครับว่าแค่นั่งมองยังฟินเลย!

## 💡 โมเมนต์เจ๋งๆ ของนักแก้ปัญหา

ระหว่างที่พัฒนาโปรเจกต์นี้อยู่นั่นเอง ความซวยก็ดันเข้าไปกระแทกเข้าให้กับ **กล่องช่องสำหรับเลือกวันที่ (Date-picker field)** อย่างจัง

โดยปกติแล้ว ขั้นตอนของระบบควรจะชิวมากๆ: ก็แค่พ่นคำสั่งสั่งบอตว่า "ลบคำ" ทิ้งไปแล้วกรอกข้อมูลวันที่อันใหม่ทับลงไปสิ! แต่เอาเข้าจริงผมดันไปคลำเจอการเกิดจุดบั๊กปริศนาครับ! เมื่อไหร่ก็ตามที่เราพยายามจะเทลบวันที่ที่เป็นค่าเบื้องต้นออก จนช่องมันเหลือเพียงความว่างเปล่า... เว็บเพจมันก็จะ Error เด้งกลายเป็นหน้าจอสีขาวโพลนไปเลยทันที

ผมไม่เคยเจอความผิดปกติแบบนี้มาก่อนเลย แต่เมื่อระดมการคิดพิจารณาสักพัก ก็เกิดอาการปิ๊งไอเดียว่า ผมควรหันไปหลอกตัว UI ซ่ะ! แทนที่จะเรียกคำสั่งแบบซื่อๆ ว่า "ลบข้อความ (Clear text)" ธรรมดา ผมก็พลิกแพลงสั่งบอตไปว่า...  **`"กดปุ่ม Command (หรือ Ctrl) + A"` (รวบเลือกทั้งหมด)** แบบรัวๆ ภายในนัดเดียว แล้วให้ตัวหุ่นยนต์จดจ่อในการกดตัวเลขวันเวลาลงไปเลยสิ เพื่อกลบความขาวของช่องซ่ะ!

และแล้ว วิธีนอกกรอบสุดเจ๋งนี้ก็ช่วยพาผมรอดจากปัญหานั้นได้สำเร็จ เว็บเพจสามารถวางเลขที่สวยงาม ข้ามขั้นตอนบั๊กป่วนไปได้อย่างสง่างามจนจบหน้าต่างครับ เหตุการณ์นี้เป็นเครื่องเตือนใจที่ดีเลยครับว่า ออโตเมชันหลายๆ ครั้งน่ะ ต้องการการพลิกแพลงคิดนอกกรอบบ้าง!

## 🧩 เจาะลึกเบื้องหลัง: การเรียบเรียงโครงสร้างขององค์รวม

เพื่อปกป้องความสะอาดตาและยกความเป็นระบบระเบียบให้มีมาตรฐาน ผมไม่ได้เหมารวมโค้ดลงมาใส่แฟ้มเดียวเหมือนกระบะทรายแบบรกๆ นะครับ แต่ผมนำโครงสร้างสายแข็งที่ชื่อว่า **Page Object Model (POM)** มาร่วมจัดการ โดยแยกย่อยเป็นสามภาคส่วน

1. 🗺️ **หน้าแผนที่หลัก (`locators.resource`)**: โฟลเดอร์หน้านี้จะเก็บรวบรวม "ที่อยู่ (Address พิกัดข้อมูล หรือ IDs/XPaths)" ของทุกวัตถุบนเพจครับ ข้อดีคือหากวันไหนลูกค้าจะขอเปลี่ยนโฉมเว็บไซต์ เราก็ตามแก้เพียงแค่หน้าโฟลเดอร์นี้จบ ไม่ต้องวิ่งหาบรรทัดให้ปวดหัว!
2. ⚙️ **ศูนย์บัญชาการชุดคำสั่ง (`keywords.resource`)**: ฐานข้อมูลนี้ทำหน้าที่เหมือนศูนย์รวบรวมเทคนิคเจาะทะลุคำสั่งยากๆ แล้วคราฟต์บรรจุใส่กล่องชุดคำสั่งสวยๆ อย่างเช่น `Fill In First Name` ครับ
3. 📝 **แผนปฏิบัติการของจริง (`test_case.robot`)**: ถัดมานี่คือสคริปต์ที่จะวิ่งงานลงไปบนสนามจริงๆ ครับ ซึ่งก็เป็นผลพวงจากสองไฟลล์ก่อนหน้าแหละครับ ที่ช่วยอัพเกรดให้โค้ดตรงหน้านี้สวยเนี้ยบ ดูง่ายๆ แต่อ่านไหลลื่นไปราวกับแผนตรวจงาน (Check-list) ระดับเทพเลย

## 🛠️ โซนเชิดชูคนสายแข็ง (For The Technical Reviewers)

ถ้ามีใครคันไม้คันมือ อยากดำดิ่งมุดถ้ำเข้าไปงัดโค้ดขึ้นมาดูเอง เชิญเสพรายละเอียดฝั่งสายเทคได้ตรงนี้เลยครับ!

### คลังแสง (Technology Stack)

- **Robot Framework**: แกนหลัก Automation framework
- **SeleniumLibrary**: ไลบรารีการจัดการระบบบนเว็บเบราว์เซอร์
- **Python**: เครื่องยนต์สำหรับขับเคลื่อนชุดคริปต์หลัก

### คำแนะนำการทำพิธีเปิดเครื่องเองบนระบบคุณ Local Run

1. คอนเฟิร์มก่อนว่าตัวเครื่องคอมคุณติดตั้ง Python ไว้แล้วรึยัง
2. สาดมนตราคำสั่งเพื่อเชิญไลบรารีที่ขลังๆ:
   ```bash
   pip install robotframework
   pip install robotframework-seleniumlibrary
   ```
3. ยืนยันพิกัดตัวไดรเวอร์ควบคุม (อย่างเช่น `chromedriver`) ในโฟลเดอร์รันงาน `PATH` ว่าอยู่ในสายรหัสหรือไม่
4. จากนั้นเตรียมชมฉากมันส์ๆ:
   ```bash
   cd form-filling-page
   robot test_case.robot
   ```

   หน้าตารายงานความพินาศหรือชัยชนะ (`log.html` และ `report.html`) จะอุบัติพ่นขึ้นมาในชั้นแฟ้มเดียวกันครับ!
